# 🚀 クイックスタートガイド

## 5 分で始める不良分析 QC7 つ道具システム

### ステップ 1: システムの起動

#### Windows の場合

1. `run.bat` ファイルをダブルクリック
2. ブラウザで `http://localhost:8501` にアクセス

#### Mac/Linux の場合

1. ターミナルで以下のコマンドを実行：

```bash
./run.sh
```

2. ブラウザで `http://localhost:8501` にアクセス

### ステップ 2: サンプルデータで試す

1. サイドバーの「📋 サンプルデータを生成」ボタンをクリック
2. 自動的にテストデータが生成されます
3. データ概要が表示されることを確認

### ステップ 3: QC7 つ道具を試す

1. サイドバーの「🔧 QC7 つ道具」から「1. パレート図」を選択
2. 不良項目別のランキンググラフが表示されます
3. 他のツールも順番に試してみてください

### ステップ 4: 実際のデータで分析

1. サイドバーの「📁 データアップロード」から CSV ファイルを選択
2. データが正しく読み込まれることを確認
3. 各 QC7 つ道具で分析を実行

## 📊 各ツールの使い方

### パレート図

- **目的**: 不良項目の重要度を把握
- **見方**: 左から右に並んだ棒グラフで、80%ライン以下の項目が重点管理対象

### 特性要因図

- **目的**: 不良原因の体系的整理
- **見方**: 4M（人・機械・材料・方法）で原因を分類

### ヒストグラム

- **目的**: 不良率の分布を把握
- **見方**: 山の形でデータのばらつきを確認

### 散布図

- **目的**: 2 つの要因の関係を分析
- **見方**: 点の分布で相関関係を確認

### 管理図

- **目的**: 工程の安定性を監視
- **見方**: 管理限界線を超えた点が異常値

### チェックシート

- **目的**: データ収集の標準化
- **見方**: 項目別のチェックリストで漏れを防止

### グラフ

- **目的**: 時系列や比較分析
- **見方**: 複数のグラフタイプで傾向を把握

## 🔧 トラブルシューティング

### よくある問題と解決方法

**Q: アプリが起動しない**

- Python 3.8 以上がインストールされているか確認
- 必要なライブラリがインストールされているか確認

**Q: データが読み込めない**

- CSV ファイルの文字エンコーディングを UTF-8 に変更
- 列名が日本語で正しく設定されているか確認

**Q: グラフが表示されない**

- ブラウザの JavaScript が有効になっているか確認
- ブラウザを最新版に更新
