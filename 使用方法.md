# 📖 詳細使用方法

## データ準備

### 必要なデータ項目

| 項目名   | 必須 | 説明               | 例         |
| -------- | ---- | ------------------ | ---------- |
| 日付     | ✅   | データの日付       | 2024-01-01 |
| 製品名   | ✅   | 製品の名称         | 製品 A     |
| 不良項目 | ✅   | 不良の種類         | 寸法不良   |
| 不良数   | ✅   | 不良の件数         | 5          |
| 検査数   | ✅   | 検査した件数       | 100        |
| 原因分類 | ⚠️   | 不良の原因         | 加工       |
| 発生工程 | ⚠️   | 不良が発生した工程 | 旋盤       |
| 備考     | ❌   | その他の情報       | 工具摩耗   |

### データ形式の例

```csv
日付,製品名,不良項目,不良数,検査数,原因分類,発生工程,備考
2024-01-01,製品A,寸法不良,5,100,加工,旋盤,工具摩耗
2024-01-01,製品A,表面粗さ,3,100,工具,旋盤,切削油不足
2024-01-02,製品B,欠け,2,80,材料,研削,材料不良
```

## 各ツールの詳細使用方法

### 1. パレート図

#### 目的

- 不良項目の重要度を視覚化
- 重点管理項目の特定
- 改善優先順位の決定

#### 使い方

1. サイドバーで「1. パレート図」を選択
2. 自動的に不良項目別のランキングが表示
3. 80%ライン以下の項目を重点管理対象として確認

#### 見方

- **縦軸（左）**: 不良数
- **縦軸（右）**: 累積比率
- **横軸**: 不良項目（多い順）
- **赤い線**: 80%ライン
- **青い棒**: 各項目の不良数

#### 活用方法

- 上位 3 項目に集中して改善活動を実施
- 月次でパレート図を更新し、改善効果を確認

### 2. 特性要因図

#### 目的

- 不良原因の体系的整理
- 4M 分析による原因分類
- 原因の可視化

#### 使い方

1. サイドバーで「2. 特性要因図」を選択
2. 4M（人・機械・材料・方法）で原因が分類表示
3. 各原因の発生件数を確認

#### 見方

- **Man（人）**: 作業者関連の原因
- **Machine（機械）**: 設備・工具関連の原因
- **Material（材料）**: 材料・部品関連の原因
- **Method（方法）**: 手順・環境関連の原因

#### 活用方法

- 原因別の対策を体系的に立案
- 定期的に原因分析を実施

### 3. ヒストグラム

#### 目的

- 不良率の分布を把握
- データのばらつきを確認
- 正規分布との比較

#### 使い方

1. サイドバーで「3. ヒストグラム」を選択
2. 不良率の分布が自動表示
3. 統計情報（平均・標準偏差・最大値）を確認

#### 見方

- **横軸**: 不良率（%）
- **縦軸**: 頻度
- **赤い線**: 平均値
- **山の形**: データの分布

#### 活用方法

- 工程能力の評価
- 改善前後の分布比較

### 4. 散布図

#### 目的

- 2 つの要因の関係を分析
- 相関関係の確認
- 原因と結果の関係把握

#### 使い方

1. サイドバーで「4. 散布図」を選択
2. X 軸と Y 軸の項目を選択
3. 相関係数を確認

#### 見方

- **横軸**: 選択した X 軸項目
- **縦軸**: 選択した Y 軸項目
- **点の分布**: 相関関係
- **色分け**: 原因分類

#### 活用方法

- 強い相関（0.7 以上）がある要因を特定
- 原因と結果の関係を定量的に把握

### 5. 管理図

#### 目的

- 工程の安定性を監視
- 異常値の検出
- 工程管理の標準化

#### 使い方

1. サイドバーで「5. 管理図」を選択
2. p 管理図（不良率管理図）が自動表示
3. 異常値の有無を確認

#### 見方

- **横軸**: 日付
- **縦軸**: 不良率（%）
- **青い線**: 実際の不良率
- **緑の線**: 中心線（平均）
- **赤い線**: 管理限界（UCL/LCL）

#### 活用方法

- 管理限界を超えた点の原因調査
- 工程の安定性評価

### 6. チェックシート

#### 目的

- データ収集の標準化
- 漏れの防止
- 作業の効率化

#### 使い方

1. サイドバーで「6. チェックシート」を選択
2. 不良項目別のチェックリストを確認
3. カスタムチェックリストを作成

#### 見方

- **不良項目別**: 各項目の詳細情報
- **カスタム**: 独自のチェック項目

#### 活用方法

- 日常のデータ収集に活用
- 検査項目の標準化

### 7. グラフ

#### 目的

- 時系列分析
- 比較分析
- 傾向把握

#### 使い方

1. サイドバーで「7. グラフ」を選択
2. グラフタイプを選択
3. 分析結果を確認

#### グラフタイプ

- **時系列グラフ**: 不良率の推移
- **工程別比較**: 工程間の比較
- **製品別比較**: 製品間の比較
- **原因別比較**: 原因別の分析

#### 活用方法

- 改善効果の確認
- トレンド分析

## データの保存・出力

### グラフの保存

1. グラフ上で右クリック
2. 「画像として保存」を選択
3. PNG 形式で保存

### データのダウンロード

1. 各ツール画面の「ダウンロード」ボタンをクリック
2. CSV 形式でデータを保存

## ベストプラクティス

### データ収集

- 毎日定期的にデータを入力
- 原因分類を統一する
- 備考欄に詳細情報を記録

### 分析の進め方

1. パレート図で重点項目を特定
2. 特性要因図で原因を整理
3. 管理図で工程の安定性を確認
4. 改善活動を実施
5. 効果をグラフで確認

### 改善活動

- 上位 3 項目に集中
- 4M 分析で原因を特定
- 管理図で効果を確認
- 継続的な改善サイクルを回す

**💡 コツ**: 定期的にデータを更新し、改善活動の効果を可視化することで、継続的な品質向上を実現できます。
